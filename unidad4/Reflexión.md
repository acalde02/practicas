# Reflexión

Para el desarrollo de la práctica opté por el Caso 1, enfocado en la construcción de un agente de atención al cliente para eCommerce. La razón principal de esta elección fue su alta aplicabilidad en entornos reales, ya que actualmente muchas empresas utilizan asistentes automatizados para resolver consultas frecuentes relacionadas con productos, disponibilidad y políticas de servicio. Consideré que este caso permitía integrar de forma más clara el uso de herramientas externas, como Google Sheets, con un modelo de lenguaje, lo cual lo hacía más representativo de un escenario profesional.

Durante el desarrollo no encontré mayores dificultades en la configuración inicial del modelo, dado que n8n facilita la integración mediante el uso de credenciales y tokens. Sin embargo, uno de los principales retos fue comprender correctamente cómo el agente interactúa con las herramientas, especialmente en la consulta del inventario en tiempo real. También fue necesario ajustar el prompt para evitar respuestas incorrectas o inventadas, así como implementar restricciones claras que limiten el comportamiento del agente. Adicionalmente, la incorporación de medidas básicas de seguridad frente a intentos de prompt injection requirió un entendimiento más cuidadoso del control del flujo de instrucciones.

Si se dispusiera de más tiempo o recursos, se podrían implementar mejoras significativas, como un sistema de búsqueda más inteligente que permita reconocer variaciones en los nombres de los productos, así como una memoria más avanzada que registre preferencias del usuario. También sería relevante integrar el agente con sistemas reales de gestión de pedidos y logística, lo que permitiría responder consultas más complejas. Otras mejoras incluirían la integración multicanal (por ejemplo, WhatsApp o web) y la incorporación de analítica para identificar patrones de uso y optimizar el servicio.

En un contexto profesional, este tipo de agentes puede aplicarse directamente en plataformas de comercio electrónico para automatizar la atención al cliente, reduciendo la carga operativa del equipo humano y mejorando los tiempos de respuesta. Además, su integración con bases de datos reales permite ofrecer información precisa y actualizada, lo cual es fundamental para la experiencia del usuario. En conjunto, esta práctica evidencia el potencial de los agentes basados en IA para resolver problemas reales de manera eficiente, siempre que se diseñen con una adecuada combinación de lógica, control y acceso a datos externos.
